Introduction

Bowerbird provides to foundation for a CSS based layout. Bowerbird is design agnostic and has been developed with the intention of project specific CSS being layered on top of it.

Borrows from Object-Orientated CSS.

Guiding Principles

Fat HTML, Thin CSS

If a developer can't read the code, it's worthless. CSS often become unreadable because of complex selectors fighting for specificity and many properties being attached to a single rule. CSS needs to be carefully looked after to ensure it remains maintainable. HTML rarely becomes unreadable – abuse it.

Inheritance Selectors Cause Fights

CSS provides developers with many options for selecting which elements to style… that doesn't mean you should use them all. Problems commonly arise when using inheritance selectors because of conflicting specificity. To prevent this happening avoid using inheritance selectors and (if possible) don't use anything except single class selectors. Following this principle means that the only thing that affects the order rules are applied is the order they are declared in.

Consistency is Key

If you want an orange background you make a .bg-orange class. If there's already a .bg-orange class defined that's a slightly different orange, you need to talk to the designer. It's likely that only one of them is correct.

Rationale

Rejected Approaches

Style Attribute

Named Elements

Blueprint CSS

Chosen Approach

  • Easier to read and with slightly less repetition than using the Style Attribute approach.
  • Far more flexible than the Named Elements approach.
  • No duplication of classes in CSS, unlike the Blueprint CSS approach.
